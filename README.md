## Investigation Workflow

1. Validate alert context and initial severity.
2. Correlate host, user, process, and time-based telemetry.
3. Extract and validate IOCs from observed activity.
4. Map behaviors to MITRE ATT&CK techniques.
5. Determine scope, likely root cause, and containment actions.
6. Document findings and escalation-ready recommendations.

## Portfolio Project Hierarchy

### 1) vSOC-Lab

Self-built virtual SOC environment for enterprise-style monitoring and investigation workflows.

- Focus areas: log ingestion, indexing, triage workflow, incident documentation
- Tooling: Splunk, Windows telemetry, Sysmon
- Outcome: repeatable detection validation and investigation process

### 2) Sigma Detection Collection

Structured detection repository organized by ATT&CK-relevant behavior.

- Focus areas: execution, persistence, and high-confidence detections
- Method: test detection logic against Windows telemetry in lab
- Outcome: practical rule tuning and false-positive analysis

### 3) Incident Response Case Studies

Documented investigations from alert to containment recommendations.

- Focus areas: triage timeline, evidence review, root cause analysis
- Cases: encoded PowerShell activity, registry run-key persistence, service creation abuse
- Outcome: clear, escalation-ready investigation records

### 4) pfSense Firewall Lab

Virtual firewall lab for traffic-control and segmentation behavior testing.

- Focus areas: policy logic, filtering outcomes, baseline hardening
- Outcome: improved understanding of network-layer detection context
